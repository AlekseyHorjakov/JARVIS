# Правила разработки

## Основные принципы

1. Один коммит — одно логическое изменение.
2. Новые папки создаются только при реальной необходимости.
3. Новые документы создаются только тогда, когда они описывают уже существующий функционал.
4. README обновляется после появления нового функционала.
5. Каждый workflow хранится отдельным JSON-файлом.
6. Каждый промпт хранится отдельным Markdown-файлом.
7. Все изменения сначала тестируются, затем попадают в репозиторий.
